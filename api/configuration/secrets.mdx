Secrets are used to store and manage sensitive information, such as passwords, OAuth tokens, and SSH keys.

## List Secrets

<ParamField query="config" type="string">
  The kubeconfig identifier to use for authentication
</ParamField>

<ParamField query="cluster" type="string">
  The cluster name to query
</ParamField>

<ParamField query="namespace" type="string">
  Filter Secrets by namespace. If not provided, returns Secrets from all namespaces
</ParamField>

### Request

```bash
curl -X GET "http://localhost:3000/api/v1/secrets?config=default&cluster=my-cluster" \
  -H "Accept: application/json"
```

### Response

<ResponseField name="uid" type="string">
  Unique identifier for the Secret
</ResponseField>

<ResponseField name="namespace" type="string">
  The namespace where the Secret is located
</ResponseField>

<ResponseField name="name" type="string">
  Name of the Secret
</ResponseField>

<ResponseField name="type" type="string">
  Type of the Secret (e.g., Opaque, kubernetes.io/service-account-token, kubernetes.io/dockerconfigjson)
</ResponseField>

<ResponseField name="keys" type="string[]">
  List of data keys stored in the Secret
</ResponseField>

<ResponseField name="data" type="integer">
  Number of data entries in the Secret
</ResponseField>

<ResponseField name="age" type="timestamp">
  Creation timestamp of the Secret
</ResponseField>

```json Response Example
[
  {
    "uid": "b2c3d4e5-f6g7-8901-bcde-fg2345678901",
    "namespace": "default",
    "name": "db-credentials",
    "type": "Opaque",
    "keys": ["password", "username"],
    "data": 2,
    "age": "2024-03-01T10:30:00Z"
  }
]
```

## Get Secret Details

<ParamField path="name" type="string" required>
  Name of the Secret
</ParamField>

<ParamField query="config" type="string">
  The kubeconfig identifier to use for authentication
</ParamField>

<ParamField query="cluster" type="string">
  The cluster name to query
</ParamField>

<ParamField query="namespace" type="string" required>
  Namespace of the Secret
</ParamField>

### Request

```bash
curl -X GET "http://localhost:3000/api/v1/secrets/db-credentials?namespace=default&config=default&cluster=my-cluster" \
  -H "Accept: application/json"
```

<Warning>
  Secret data values are base64 encoded in the response. Handle sensitive information with care.
</Warning>

## Get Secret YAML

Retrieve the full Kubernetes manifest in YAML format.

### Request

```bash
curl -X GET "http://localhost:3000/api/v1/secrets/db-credentials/yaml?namespace=default&config=default&cluster=my-cluster" \
  -H "Accept: application/yaml"
```

## Get Secret Events

Retrieve Kubernetes events related to the Secret.

### Request

```bash
curl -X GET "http://localhost:3000/api/v1/secrets/db-credentials/events?namespace=default&config=default&cluster=my-cluster" \
  -H "Accept: application/json"
```

## Delete Secret

<ParamField query="name" type="string" required>
  Name of the Secret to delete
</ParamField>

<ParamField query="namespace" type="string" required>
  Namespace of the Secret
</ParamField>

<ParamField query="config" type="string">
  The kubeconfig identifier to use for authentication
</ParamField>

<ParamField query="cluster" type="string">
  The cluster name
</ParamField>

### Request

```bash
curl -X DELETE "http://localhost:3000/api/v1/secrets?name=db-credentials&namespace=default&config=default&cluster=my-cluster"
```

### Response

Returns HTTP 200 on successful deletion.
