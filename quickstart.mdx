Before you begin, ensure you have:

  <Card title="Kubernetes cluster" icon="server">

## Installation

Choose your preferred installation method:

<Steps>
  <Step title="Install Kubewall">
    Select the installation method that works best for your environment:

    <Tabs>
      <Tab title="Docker">
        ```bash
        docker run -p 7080:7080 -v kubewall:/.kubewall ghcr.io/kubewall/kubewall:latest
        ```
      </Tab>
      <Tab title="Homebrew">
        ```bash
        brew install --cask kubewall/tap/kubewall
        ```
      </Tab>
      <Tab title="Binary">
        Download the latest binary from the [releases page](https://github.com/kubewall/kubewall/releases) and add it to your PATH.
      </Tab>
    </Tabs>

    <Info>
    See the [Installation](/installation) page for all available installation methods.
    </Info>
  </Step>

  <Step title="Start Kubewall">
    If you installed using a package manager or binary, start Kubewall:

    ```bash
    kubewall
    ```

    You should see the Kubewall banner:

    ```
     _          _                        _ _ 
    | | ___   _| |__   _____      ____ _| | |
    | |/ / | | | '_ \ / _ \ \ /\ / / _` | | |
    |   <| |_| | |_) |  __/\ V  V / (_| | | |
    |_|\_\\__,_|_.__/ \___| \_/\_/ \__,_|_|_|
    ___________________________________________
    version: x.x.x
    commit: xxxxxxx
    https://github.com/kubewall/kubewall
    ___________________________________________
    ```

    <Note>
    By default, Kubewall automatically opens your default browser. Use `--no-open-browser` to disable this behavior.
    </Note>
  </Step>

  <Step title="Access the dashboard">
    Open your browser and navigate to:

    ```
    http://localhost:7080
    ```

    The Kubewall dashboard will load, displaying your connected Kubernetes clusters.
  </Step>
</Steps>

## Connecting to clusters

Kubewall automatically detects clusters from your kubeconfig file (typically located at `~/.kube/config`).

<Accordion title="How cluster detection works">
  Kubewall reads your kubeconfig file and displays all configured contexts. You can switch between clusters using the cluster selector in the dashboard.

  If you have multiple kubeconfig files, you can set the `KUBECONFIG` environment variable before starting Kubewall:

  ```bash
  export KUBECONFIG=~/.kube/config:~/.kube/prod-config
  kubewall
  ```
</Accordion>

## Common operations

### Viewing pod logs

<Steps>
  <Step title="Navigate to Workloads > Pods">
    Select your namespace and find the pod you want to inspect.
  </Step>
  <Step title="Click on the pod name">
    This opens the pod detail view with real-time information.
  </Step>
  <Step title="View logs">
    Click the logs tab to stream logs from the pod's containers with advanced search and tail options.
  </Step>
</Steps>

### Port forwarding

Access services running inside your cluster on your local machine:

<Steps>
  <Step title="Navigate to the service or pod">
    Find the resource you want to port forward to.
  </Step>
  <Step title="Click the port forward button">
    Select the port you want to forward.
  </Step>
  <Step title="Access locally">
    The service is now accessible on your local machine at `localhost:<port>`.
  </Step>
</Steps>

<Tip>
No complex CLI commands or YAML edits required—just one click!
</Tip>

### Applying manifests

Deploy or update resources using YAML manifests:

<Steps>
  <Step title="Click the Apply Manifest button">
    Usually found in the top navigation or resource list view.
  </Step>
  <Step title="Paste or edit your YAML">
    Use the built-in Monaco editor with syntax highlighting and validation.
  </Step>
  <Step title="Apply the changes">
    Review and apply the manifest to your cluster.
  </Step>
</Steps>

## Configuration options

Kubewall supports several command-line flags for customization:

<ResponseField name="--listen" type="string" default="[::]:7080">
  IP and port to listen on (e.g., `localhost:7080`, `:7080`, or `[::]:7080`)
</ResponseField>

<ResponseField name="--certFile" type="string">
  Absolute path to certificate file for HTTPS
</ResponseField>

<ResponseField name="--keyFile" type="string">
  Absolute path to key file for HTTPS
</ResponseField>

<ResponseField name="--k8s-client-qps" type="int" default="100">
  Maximum QPS to the Kubernetes API server
</ResponseField>

<ResponseField name="--k8s-client-burst" type="int" default="200">
  Maximum burst for throttle when communicating with the Kubernetes API
</ResponseField>

<ResponseField name="--no-open-browser" type="boolean" default="false">
  Do not automatically open the default browser on startup
</ResponseField>

### Example: Custom configuration

```bash
kubewall \
  --listen localhost:8080 \
  --k8s-client-qps 200 \
  --k8s-client-burst 400 \
  --no-open-browser
```
